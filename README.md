# Course Adapt

## Описание проекта

Course Adapt - веб-приложение для создания учебных курсов, генерации тестов и отслеживания персонального прогресса обучения.

## Возможности

- регистрация, вход и выход из учетной записи;
- создание курсов с темами, описанием, указаниями для генерации и начальной сложностью;
- публикация курсов в общем каталоге и добавление публичных курсов себе;
- генерация и прохождение персональных тестов;
- просмотр результатов тестов и истории прохождения;
- редактирование профиля пользователя;
- административное управление пользователями и публичным доступом к курсам.

## Структура проекта

Проект разделен на серверную часть, клиентскую часть и общий слой запуска.

```text
backend/              
  ai/                 интеграция с LLM, шаблоны и контракты генерации
  cli/                модуль cli-скриптов
  core/               конфиг, DI, настройки бд, auth, кэш, логи
  handlers/           HTTP-ручки FastAPI
  migration/          миграции
  models/             SQLAlchemy-модели
  repositories/       слой работы с бд
  schemas/            Pydantic-схемы
  seed/               начальные данные
  services/           бизнес-логика
  tests/              тесты

frontend/             
  nginx/              конфиг nginx для сборки
  src/                
    api/              клиентские запросы и типы API
    assets/           стили и статические ресурсы
    components/       переиспользуемые Vue-компоненты
    composables/      общая логика
    router/           роутинг
    stores/           хранилища
    utils/            вспомогательные функции
    views/            страницы приложения

logs/                 локальные файловые логи бэка
```

## Введение в эксплуатацию

### Требования

Для запуска проекта нужны Docker, Docker Compose и `make`.

### Настройка окружения

Перед первым запуском нужно создать env-файлы из шаблонов:

```bash
cp .env.example .env
cp .env.test.example .env.test
```

В `.env` указываются настройки основного приложения, в `.env.test` - настройки тестового окружения.

### Запуск приложения

Проект поднимается из корня одной командой:

```bash
make up
```

После запуска frontend будет доступен по адресу:

```text
http://localhost:8080
```

Backend будет доступен по адресу:

```text
http://localhost:8000
```

Если нужно остановить проект:

```bash
make down
```

Перезапуск выполняется так:

```bash
make restart
```

### Миграции и начальные данные

При обычном запуске миграции и начальные данные применяются автоматически. Если нужно выполнить эти шаги отдельно, можно использовать:

```bash
make migrate
make seed
```

Или одной командой:

```bash
make init
```

### Логи

Логи контейнеров доступны через:

```bash
make logs
```

Backend пишет файл логов в:

```text
logs/backend/backend.log
```

### Тесты

Полная проверка проекта запускается командой:

```bash
make test
```

Backend и frontend можно проверить отдельно:

```bash
make test-backend
make test-frontend
```

### Команды Makefile

```text
make build          собрать образы
make build-backend  собрать backend-образ
make up             поднять приложение
make down           остановить приложение
make restart        перезапустить приложение
make logs           смотреть логи
make migrate        применить миграции
make seed           загрузить начальные данные
make init           выполнить migrate и seed
make test           запустить все проверки
make test-backend   запустить backend-тесты
make test-frontend  запустить frontend-проверки
make test-down      остановить тестовый контур
make clean          удалить контейнеры и volumes
```

## Руководство пользователя

### Авторизация и регистрация

При открытии приложения пользователь попадает в интерфейс сервиса. Если пользователь не авторизован, ему доступны экран входа, экран регистрации и публичный каталог курсов. На экране входа можно ввести email и пароль существующей учетной записи. На экране регистрации можно создать новый аккаунт, указав имя пользователя, email, пароль и, при необходимости, описание профиля.

После успешного входа приложение открывает основную область системы. Если пользователь пытается выполнить действие, требующее авторизации, например добавить публичный курс себе или открыть личный раздел, система перенаправляет его к входу или регистрации.

### Главный экран

Главный экран показывает краткую сводку по обучению. Здесь отображаются активные курсы, количество тестов за неделю, число добавленных и публичных курсов, а также блок последнего завершенного теста. В блоке последнего теста можно увидеть процент правильных ответов, количество вопросов и ошибок. Если тест уже завершен, из этого блока можно перейти к просмотру разбора.

Ниже располагается список курсов пользователя и переход в публичный каталог. Карточка курса открывает детальную страницу курса.

### Мои курсы

В разделе своих курсов отображаются курсы, которые пользователь создал сам или добавил из публичного каталога. Для каждого курса показываются название, описание, статус публичности и дата создания. Из списка можно перейти в детальный просмотр курса.

Если курс был создан пользователем, ему доступны действия владельца: изменение публичности и удаление курса. Если курс был добавлен из каталога, пользователь может убрать его из своего списка. При этом тесты и прогресс остаются индивидуальными для каждого пользователя.

### Создание курса

На экране создания курса пользователь заполняет название, описание, указания для генерации, список тем, начальную сложность и признак публичности. Описание используется как обычный комментарий к курсу. Указания для генерации передаются AI-модели и влияют на структуру курса и будущие тесты.

Темы добавляются отдельными элементами. После отправки формы система создает курс, формирует его структуру и задает начальный прогресс по подтемам с учетом выбранной сложности. Если курс отмечен как публичный, он появляется в общем каталоге и становится доступен другим пользователям для добавления.

### Детальный просмотр курса

На странице курса отображаются основные сведения: название, описание, указание для генерации, темы и история тестов. При наведении на тему можно увидеть связанные подтемы и их прогресс.

Из детального просмотра можно создать новый тест, открыть уже созданный тест, продолжить незавершенное прохождение или посмотреть разбор завершенного теста. Если последний тест по курсу еще не завершен, создание нового теста блокируется, чтобы пользователь сначала завершил текущее прохождение.

### Прохождение теста

Экран прохождения теста содержит список вопросов и варианты ответов. В тесте могут быть вопросы с одним правильным вариантом и вопросы с множественным выбором. Пользователь выбирает ответы и отправляет результат на проверку.

После успешной отправки кнопка отправки блокируется, чтобы результат не был отправлен повторно. Если пользователь вышел из приложения или покинул страницу до завершения теста, незавершенный тест можно открыть снова и продолжить прохождение.

### Просмотр результатов теста

Экран просмотра результатов показывает итог прохождения теста. Пользователь видит вопросы, свои выбранные варианты и информацию о корректности ответа. Для вопросов с множественным выбором ответ считается правильным только при полном совпадении выбранного набора вариантов с правильным набором.

Разбор доступен только для завершенных тестов. Если тест находится в процессе прохождения, система предлагает продолжить тест вместо открытия результатов.

### Публичный каталог

В публичном каталоге отображаются курсы, опубликованные другими пользователями. Каталог можно просматривать без авторизации. Поиск выполняется по ключевым словам, также доступна сортировка по дате создания.

Неавторизованный пользователь может открыть список и детальную страницу публичного курса, но для добавления курса себе потребуется войти или зарегистрироваться. Авторизованный пользователь может добавить публичный курс в свой список и выбрать начальную сложность. После добавления для него создается персональный прогресс, а дальнейшие тесты и результаты прохождения хранятся отдельно от других пользователей.

### Профиль

В профиле отображаются email, имя пользователя и описание профиля. Email и имя пользователя доступны только для просмотра. Описание профиля можно добавить, изменить или удалить.

В этом же разделе находится команда выхода из системы. При выходе сервер отзывает refresh-токены пользователя, а клиент очищает состояние текущей сессии.

### Административный раздел

Пользователь с ролью администратора получает доступ к дополнительным функциям. В интерфейсе просмотра курсов администратор может снять публичный курс с общего доступа и заблокировать пользователя. Если курс был ограничен администратором, владелец курса не сможет самостоятельно вернуть его в публичный каталог.

В административном разделе отображаются заблокированные пользователи и курсы, снятые с публичного доступа. Здесь администратор может разблокировать пользователя или вернуть курсу возможность публикации.
